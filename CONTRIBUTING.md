# Contributing to AwesomeMCP

First off, thank you for taking the time to contribute! It is contributions like yours that make the MCP ecosystem grow.

When contributing, please ensure your pull request adheres to the following guidelines.

## Contribution Guidelines

### 1. Quality Standards
- **Functional:** The MCP server must be functional and include a clear `README` with installation instructions.
- **Categorization:** Place the server in the most appropriate category. If a category doesn't exist, feel free to suggest one.
- **Link Formatting:** Use the format: `* [Name](URL) - Short, descriptive summary.`
- **Check for Duplicates:** Before submitting, please search the list to ensure the server isn't already listed.

### 2. Formatting Requirements
- Descriptions should start with an uppercase letter and end with a period.
- Avoid "marketing" speak (e.g., "The best server ever"). Stick to what it actually *does*.
- Ensure the link leads directly to the repository or the specific project page.

### 3. Submission Process
1. **Fork** the repository.
2. **Create a new branch** for your suggestion (e.g., `feat/add-my-server`).
3. **Add the server** to the `README.md` in alphabetical order within its category.
4. **Commit your changes** with a clear message (e.g., `Add [Server Name] to Developer Tools`).
5. **Open a Pull Request** and provide a brief explanation of why this server belongs on the list.

## Style Guide for New Entries

Please follow this template for consistency:

```markdown
* [Server Name](https://github.com/user/repo) - A brief description of the server's primary function and supported tools.
```

## Maintenance
If you find a link that is broken or a repository that is no longer maintained, please open an issue or submit a PR to remove it.
